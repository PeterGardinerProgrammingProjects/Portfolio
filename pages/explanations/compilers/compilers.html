What are Compilers?

Compilers are what turn human readable source code into instructions a computer can understand and run. Technically a compiler is just one part of this process, but that is not important for now. A compiler follows many steps, and many will be skipped. The first step is tokenization, the process of breaking down the code into its component pieces. For example, the code for(int i = 0; i < length; i++) has the tokens ["for", "(", "int", "i", "=", "0", ";",...] (at this point the tokens are just numbers instead of characters). The compiler then gives these ordered tokens to a parser, which checks the validity of the code. If the parser gets the nonsense tokens ["for", ")", "0", "while",...], it will give a compiler error, which means the code was invalid. If it gets valid tokens, it puts them into groups for the next steps. For example, the code a + b * c should be interpreted as a + (b * c) and not (a + b) * c. This is what the parser does. There are too many steps to list here, but eventually the compiler turns this data into machine instructions which the computer can understand. In fact, compilers do more than just translating, they also optimize. This means that they can analyze how code will run and change it to produce the same result in less steps. For example, this statement can always be ignored: if (2 + 2 == 5) { DoFunction(); }. There is a lot more to compiler optimizations, too much to explain here.
